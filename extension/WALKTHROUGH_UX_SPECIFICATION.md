## 5. Responsive Behavior

### Desktop (>1024px)
- Tooltip: Right of target, max-width 400px
- Overlay: Full viewport, 60% opacity
- Debug panel: Side panel, 400px width

### Tablet (768-1024px)
- Tooltip: Bottom of target, full-width
- Overlay: 70% opacity for better visibility
- Debug panel: Bottom sheet, 50% height

### Mobile (<768px)
- Tooltip: Bottom sheet, 60% height, draggable handle
- Overlay: Target at top 40% of screen
- Progress: Bottom bar, always visible
- Debug: Modal overlay, full-screen

## 6. Accessibility Requirements

### ARIA
- Tooltip has role="dialog" aria-modal="true"
- Instructions have aria-live="polite"
- Errors have aria-live="assertive"
- Target element maintains original tabindex

### Keyboard Navigation (When not blocking)
- Escape: Request abort (with confirmation)
- Tab: Move focus within tooltip only
- Enter/Space: Activate focused button

### Screen Readers
- Announce step changes: "Step 3 of 8: Save Your Changes"
- Describe target location: "Target: button, Save, top right"
- Read validation feedback immediately
